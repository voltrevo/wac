# wacParse + wacResolve

## wacParse (committed earlier this session)

Built the recursive descent parser (`wacParse.ts`) for the wac language. Consumes
tokens from wacLex, produces a `Program` AST with full error recovery.

Also added the `block` statement variant to `ast.ts` (needed for `case 0: { }`).

118 tests in wacParse.test.ts cover all 100% branch/function/line.

Key disambiguation rules:
- Uppercase-first ident `.method(` → `call` with `typeQual` (static method call)
- Lowercase ident `.method(` → `method` via postfix parsing
- Named type followed by ident → var decl (via `looksLikeVarDecl`)

## wacResolve

Built `wacResolve.ts`: resolves the import graph starting from an entry file,
returning a flat `ResolvedModule` with all declarations merged and function names
mangled (`fileStem$funcName`).

**Design:**
- Three-phase: load (DFS cycle-safe), flatten, env-build
- Placeholder entry inserted before recursion → handles circular imports
- Per-file `NameEnv` maps local names → mangled names for the type checker
- `CompileError` type (matching the spec's format: file, phase, span, annotation, hint)

**Handles:**
- Multi-file imports with aliases
- Diamond imports (each file loaded at most once)
- Circular imports
- Struct imports (struct names globally unique, no mangling)
- Lex/parse error promotion with file+phase added
- Duplicate struct/function detection
- Unknown export name errors
- File read failures

24 tests, 100% branch/function/line coverage.
