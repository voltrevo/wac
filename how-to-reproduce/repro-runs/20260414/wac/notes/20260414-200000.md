# wacInstance ‚Äî runtime wasm wrapper

Built `atoms/wac/wacInstance.ts` ‚Äî instantiates compiled wasm and exposes
each exported function as a typed JS callable.

## What it does

Takes a `WacCompiled` (from `wacCompile`) and a `Cap` with `WebAssembly.instantiate`,
and returns a `WacInstance` ‚Äî a record of wrapped functions.

Supported types: i32, i64, f32, f64, bool, void.
Unsupported types (string, array, struct, nullable, funcref): those exports are silently
skipped. Array/string support would require wasm-side marshal helpers.

## Type conversions

- bool params: JS boolean ‚Üí wasm i32 (true‚Üí1, false‚Üí0)
- bool returns: wasm i32 ‚Üí JS boolean (0‚Üífalse, nonzero‚Üítrue)
- i64: JS bigint ‚Üî wasm i64 (JS numbers are coerced via BigInt.trunc)
- void returns: JS undefined
- i32/f32/f64: pass through unchanged

## Confidence

- 16 tests covering i32/i64/f32/f64/bool/void, struct skip, array skip, cap injection
- Independently verified values: gcd(48,18)=6, fib(20)=6765, fact(10)=3628800
- 89.7% branch / 100% function / 97.5% line coverage
- Single uncovered line: defensive `typeof raw !== "function"` guard (unreachable in practice)
