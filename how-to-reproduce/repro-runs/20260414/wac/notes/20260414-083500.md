## wacLex — lexer atom

Built `atoms/wac/wacLex.ts` — the lexer for the wac language. Takes source text,
returns a flat token array plus any lex errors.

### What it does
- Lexes all 33 keywords, cast operators (`as!`, `as~`, `as@`), all 40+ operators
  and punctuation, integer/float/hex literals, string and char literals
- Tracks line and column for every token (1-based)
- Emits `LexError` entries rather than throwing, so compilation continues past errors
- `as!`, `as~`, `as@` are emitted as single tokens when the suffix immediately follows `as`
- `from`, `anyref`, `i31ref` are not in the keyword list — lexed as identifiers
- EOF sentinel token always present at end

### Confidence
38 tests, 100% branch/line/function coverage via `deno coverage`. Tests cover:
- All keywords individually
- All operator variants including compound assignment and shifts
- Disambiguation: `!=` vs `!`, `as!` vs `as`, `++`/`--` vs `+`/`-`
- Numeric literals: decimal, hex, float, edge case `3.x` (dot not a float)
- String escapes, unterminated strings (newline and EOF), unknown escapes
- Char literals: ASCII, escapes, emoji surrogate pairs, empty, unterminated
- Source location tracking verified line+col
- Error recovery: continues lexing after bad character
