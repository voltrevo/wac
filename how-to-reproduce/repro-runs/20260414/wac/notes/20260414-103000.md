# wasmBuildBin — wasm binary assembler

Built `atoms/wac/wasmBuildBin.ts` — assembles a complete `.wasm` binary from a
TypedModule using WasmGC binary format.

## What it does

Takes the output of `wacTypeCheck` and produces a `WacCompiled` with:
- A `.wasm` binary (valid WebAssembly GC module)
- A `WacExport[]` metadata array for the host binding layer

The binary includes:
1. Type section with a single rec group for all GC types (structs + arrays)
2. Function section
3. Export section (entry-file `export` functions only)
4. Code section (bodies from `wacEmitFunc`)

## Key design decisions

**Struct encoding**: Parent structs are encoded as open sub types (`0x50 0x00 0x5F ...`) so children can extend them. Leaf structs are final types (`0x5F ...`). Children are `0x50 0x01 <parentIdx> 0x5F ...`.

**WasmGC field indexing** (critical finding): V8 requires that `struct.get/set` use the *declaring* struct's type index and own field index — not the absolute position across the inheritance chain. Shape[x,y] + Rect extends Shape[w,h]: `struct.get $Rect 0` = w (NOT x). Fixed by adding `declaringStruct` and `ownFieldIdx` to the `structAllFields` return type.

**funcIdx resolution**: wacEmitFunc looks up function names as they appear in the AST (local names), while wacResolve uses mangled names (`stem$funcName`). Built per-function funcIdx maps via `NameEnv` translations.

## Confidence

- 30 tests, instantiated and executed as real wasm modules
- factorial, mutual recursion, void, structs, inheritance, methods, arrays, strings, funcrefs all verified
- 95.7% branch / 100% function / 98.1% line coverage
- Remaining uncovered: dead code (defensive guards) and impractical paths (uleb multi-byte, error throws)
