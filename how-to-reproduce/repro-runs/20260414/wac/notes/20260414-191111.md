# wacCompile + spec diagnostic fixes

Built `atoms/wac/wacCompile.ts` — thin pipeline orchestrator that chains
wacResolve → wacTypeCheck → wasmBuildBin and returns `WacCompileResult`.

Also fixed several spec diagnostic bugs in wacParse.ts and wacTypeCheck.ts
discovered while writing wacCompile's spec-tag tests.

## What was built

### wacCompile.ts

Single function `wacCompile(cap, entryPath)` returning:
- `{ ok: true, wasm, exports }` on success
- `{ ok: false, errors }` on any failure

Errors stop the pipeline: lex/parse errors → no typecheck; typecheck errors → no codegen.

### Spec diagnostic fixes

**wacParse.ts:**
- `parseCast()`: Cast AST node now anchors to the operand's position (not the
  operator token), so errors point to the start of the full cast expression.
- `expect(";")`: Reports at the end of the last token (e.g. col=16 for `5 + 2`),
  not the start of the next statement. Message split into `"expected ';'"` +
  annotation `"expected ';' after statement"`.

**wacTypeCheck.ts:**
- Added `exprSpan(expr)` helper: approximate source span for expressions (used
  for correct error underlines on float/int/ident/binary/cast expressions).
- Added `lvalSpan(lval)` helper: span of an lvalue including field chains.
- Added `prettyExpr(expr)` helper: minimal pretty-printer for hint generation.
- `var` assignment errors now point to the RHS init expression, not the variable
  name (fixes `[§wac-diag-assign-j3qm7xf]`).
- Nullable-to-non-null assignment: specific message "cannot assign nullable to
  non-null" with hint "unwrap with `!`: T name = q!;" (fixes `[§wac-diag-null-h6kp9wn]`).
- Const write errors: message changed to "cannot write through const reference",
  span covers full lval (e.g. "p.x" = 3) (fixes `[§wac-diag-const-w2jm5xf]`).
- Cast errors: span covers full "operand op type" expression (fixes `[§wac-diag-cast-p5fn2rk]`).
- Return type mismatch: bool-to-int hint added (fixes `[§wac-diag-wide-k4rn8wp]`).
- Return span now uses `exprSpan(stmt.value)` instead of hardcoded 7.
- Added `validateType()`: detects unknown struct type names in var declarations,
  emits `"expected type"` + `annotation: "unknown type 'foo'"` (fixes `[§wac-diag-parse-bad-type-n7qm3xf]`).
